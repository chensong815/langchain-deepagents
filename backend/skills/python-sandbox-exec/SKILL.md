# Python Sandbox Exec

## 适用场景

- 用户明确要求“运行 Python / 执行代码 / 验证结果 / 算一下”。
- 问题需要精确计算、数据清洗、格式转换、批量处理或解析结构化文本。
- 需要通过实际运行代码验证某个结论，而不是仅凭推断回答。
- 需要生成临时文件或临时脚本。

## 工作流程

1. 如果直接回答即可满足需求，不要为了展示能力而调用工具。
2. 需要运行代码时，优先使用 `run_python_code`，并在代码中显式 `print(...)` 输出关键结果。
3. 当前 sandbox 默认复用后端正在使用的 Python 环境。像画折线图这类常见任务，先直接写代码运行，必要时先 `import` 验证，不要先执行安装依赖。
4. 若问题依赖第三方库，先判断部署容器是否已预装；未预装时不要假设可以在运行时安装。
5. 代码应尽量短、小、可复现，优先使用标准库或容器内已提供的依赖；绘图优先直接生成图片、HTML 或 SVG 文件并给出路径。
6. 读写、删除、重命名等文件访问只允许发生在当前会话 sandbox 内；不要尝试读取或改动宿主项目目录、启动子进程或发起网络请求。
7. 回答时先给执行结果，再给简短解释；不要把未执行的推断表述成“已验证”。

## 结果约束

- 只有在工具返回 `ok=true` 时，才声称代码已成功运行。
- 如果执行失败，明确说明失败点，并基于 `stderr`/`stdout` 给出下一步处理。
- 不要凭心算替代应当通过代码验证的计算任务。
