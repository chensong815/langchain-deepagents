# 调研规划

## 适用场景
- 用户要求做调研、对比分析或趋势总结。
- 在给出结论前需要先收集事实依据。

## 工作流程
1. 用户需求不清晰时，先明确范围和输出格式。
2. 将主题拆分为 2-5 个可验证的关键问题。
3. 针对每个问题收集证据与事实。
4. 基于证据综合结论，并明确假设与置信度。
5. 用精炼要点给出结果和下一步建议。

## 输出检查清单
- 结论必须与证据一一对应。
- 对未知项和不确定性要明确标注。
- 优先给出可执行建议，避免空泛表述。
